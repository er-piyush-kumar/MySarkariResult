Leetcode 2529: Maximum Count of Positive and Negative Integers

Given a sorted array of integers, we need to find the maximum count between positive and negative integers. The problem is straightforward but teaches efficient ways to handle sorted arrays. Let's dive into different approaches to solve it!

Problem Link: LeetCode 2529

Problem Explanation

We are given an array nums sorted in non-decreasing order. We need to return the maximum value between the count of positive integers and negative integers. Note that zeros are not counted.

Example 1

Input: nums = [-2,-1,-1,1,2,3]
Output: 3
Explanation: There are 3 positive and 3 negative integers. The maximum is 3.

Example 2

Input: nums = [-3,-2,-1,0,0,1,2]
Output: 3
Explanation: 2 positives and 3 negatives. Maximum is 3.

Approach 1: Brute Force (Linear Search)

Iterate through the array and count negative and positive numbers separately. Return the maximum of the two counts.

Steps

  1. Initialize counters for negatives and positives.
  2. Loop through each element:
    • If element < 0: increment negatives.
    • If element > 0: increment positives.
  3. Return the maximum of the two counters.

C++ Code


class Solution {
public:
    int maximumCount(vector& nums) {
        int neg = 0, pos = 0;
        for (int num : nums) {
            if (num < 0) neg++;
            else if (num > 0) pos++;
        }
        return max(neg, pos);
    }
};

Time Complexity

O(n) – We traverse the array once.

Space Complexity

O(1) – Constant space for counters.

Approach 2: Binary Search (Optimized)

Since the array is sorted, use binary search to find the first non-negative and first positive index. Compute counts from these indices.

Steps

  1. Use upper_bound to find the first index where elements are > 0. The count of positives is n - pos_index.
  2. Use lower_bound to find the first index where elements are >= 0. The count of negatives is this index.
  3. Return the maximum of the two counts.

C++ Code


class Solution {
public:
    int maximumCount(vector& nums) {
        int n = nums.size();
        int firstPos = upper_bound(nums.begin(), nums.end(), 0) - nums.begin();
        int firstNonNeg = lower_bound(nums.begin(), nums.end(), 0) - nums.begin();
        int pos = n - firstPos;
        int neg = firstNonNeg;
        return max(pos, neg);
    }
};

Time Complexity

O(log n) – Binary search operations are logarithmic.

Space Complexity

O(1) – No extra space used.

Comparison of Approaches

Approach Time Complexity Space Complexity Best Use Case
Brute Force O(n) O(1) Small input sizes
Binary Search O(log n) O(1) Large sorted arrays

Frequently Asked Questions (FAQs)

1. Why is the array sorted?

The sorted property allows us to use efficient algorithms like binary search, reducing time complexity from O(n) to O(log n).

2. How does binary search work here?

upper_bound finds the first element > 0, and lower_bound finds the first element ≥ 0. The difference gives counts.

3. What if all elements are negative?

All elements will contribute to the negative count, and positives will be 0. The maximum is the negative count.

4. How are zeros handled?

Zeros are ignored since the problem specifies counting only positive and negative integers.

5. Can there be multiple zeros?

Yes, but they don't affect the result as they are skipped in counting.

6. What's the edge case for this problem?

When the array has all positives or all negatives. The code handles these by returning the respective counts.

7. Why use upper_bound and lower_bound?

upper_bound(0) gives the first index after negatives, and lower_bound(0) gives the first non-negative index.

8. What's the follow-up challenge?

Solving in O(log n) time, which is achieved using binary search.

9. Can we solve this without binary search?

Yes, using a linear scan, but it’s less efficient for large arrays.

10. Is this problem similar to finding the pivot in a sorted array?

Yes, both involve finding transition points in sorted data, making binary search ideal.
