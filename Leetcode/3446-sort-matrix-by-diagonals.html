Leetcode 3446 | Sort Matrix by Diagonals

Sorting a square matrix based on its diagonals is an interesting problem that tests our understanding of array manipulation and sorting techniques. In this article, we’ll explore three approaches to solve this problem: a Brute Force method, an Optimized Approach using a hash map, and an In-Place Sorting approach. We’ll also analyze the time complexity of each solution and answer some frequently asked questions.

Problem Link : Sort Matrix by Diagonals

Understanding the Problem

We are given an n × n square matrix, and we need to sort its diagonals according to the following rules:

  • Diagonals in the bottom-left triangle (including the middle diagonal) should be sorted in non-increasing order (descending).
  • Diagonals in the top-right triangle should be sorted in non-decreasing order (ascending).

Example 1


Input:
grid = [[1, 7, 3],
        [9, 8, 2],
        [4, 5, 6]]

Output:
grid = [[8, 2, 3],
        [9, 6, 7],
        [4, 5, 1]]

Example 2


Input:
grid = [[0, 1],
        [1, 2]]

Output:
grid = [[2, 1],
        [1, 0]]

Brute Force Approach – Sorting Each Diagonal Individually

The brute force approach involves extracting all diagonals from the matrix, sorting them, and then placing them back into the matrix.

Step-by-Step Approach
  1. Extract Diagonals: Iterate through the matrix and collect elements for each diagonal.
  2. Sort Diagonals:
    • Sort diagonals in the bottom-left triangle in descending order.
    • Sort diagonals in the top-right triangle in ascending order.
  3. Place Diagonals Back: Reconstruct the matrix with the sorted diagonals.

Time Complexity
  • Extracting diagonals: O(n²)
  • Sorting each diagonal: O(n log n)
  • Placing them back: O(n²)
  • Total Complexity: O(n² log n)

C++ Implementation


#include <vector>
#include <algorithm>
#include <iostream>

using namespace std;

class Solution {
public:
    vector<vector<int>> sortMatrix(vector<vector<int>> &grid) {
        int n = grid.size();

        // Sort all diagonals separately
        for (int d = -(n - 1); d <= (n - 1); d++) {
            vector<int> diag;

            // Collect diagonal elements
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    if (i - j == d) {
                        diag.push_back(grid[i][j]);
                    }
                }
            }

            // Sort the diagonal accordingly
            if (d >= 0)
                sort(diag.begin(), diag.end(), greater<int>()); // Descending order
            else
                sort(diag.begin(), diag.end()); // Ascending order

            // Put sorted elements back
            int index = 0;
            for (int i = 0; i < n; i++) {
                for (int j = 0; j < n; j++) {
                    if (i - j == d) {
                        grid[i][j] = diag[index++];
                    }
                }
            }
        }
        return grid;
    }
};

int main() {
    Solution sol;
    vector<vector<int>> grid = {{1, 7, 3}, {9, 8, 2}, {4, 5, 6}};

    vector<vector<int>> sortedGrid = sol.sortMatrix(grid);

    for (auto row : sortedGrid) {
        for (int num : row) {
            cout << num << " ";
        }
        cout << endl;
    }
    return 0;
}

Drawbacks of the Brute Force Approach
  • Iterating over all elements multiple times increases redundancy.
  • Sorting each diagonal separately is inefficient.

Optimized Approach – Using Hash Map for Efficient Sorting

The optimized approach uses a hash map to store and sort diagonals efficiently.

Steps to Optimize
  1. Store Diagonals in a Hash Map:
    • Key: (i - j) (unique identifier for each diagonal).
    • Value: List of elements in that diagonal.
  2. Sort Diagonals:
    • Sort diagonals in the bottom-left triangle in descending order.
    • Sort diagonals in the top-right triangle in ascending order.
  3. Reconstruct the Matrix: Place the sorted diagonals back into the matrix.

Time Complexity
  • Collecting diagonals: O(n²)
  • Sorting diagonals: O(n log n)
  • Reconstructing matrix: O(n²)
  • Total Complexity: O(n² log n), but with reduced redundancy.

C++ Implementation


#include <vector>
#include <algorithm>
#include <unordered_map>

using namespace std;

class Solution {
public:
    vector<vector<int>> sortMatrix(vector<vector<int>> &grid) {
        int n = grid.size();
        unordered_map<int, vector<int>> diagMap;

        // Collect all diagonals
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                diagMap[i - j].push_back(grid[i][j]);
            }
        }

        // Sort diagonals based on their positions
        for (auto &[key, values] : diagMap) {
            if (key >= 0) {
                sort(values.begin(), values.end(), greater<int>()); // Descending for bottom-left
            } else {
                sort(values.begin(), values.end()); // Ascending for top-right
            }
        }

        // Fill sorted values back into the matrix
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                grid[i][j] = diagMap[i - j].back();
                diagMap[i - j].pop_back();
            }
        }

        return grid;
    }
};

Approach 3: In-Place Diagonal Sorting

This approach sorts each diagonal directly within the matrix without using extra space. It identifies the starting points of diagonals and sorts them in-place.

Steps to Implement
  1. Identify Diagonals:
    • For the bottom-left triangle, diagonals start from the first column (grid[i][0]).
    • For the top-right triangle, diagonals start from the first row (grid[0][j]).
  2. Sort Each Diagonal:
    • Extract elements along the diagonal, sort them, and place them back into the matrix.
  3. Handle Sorting Order:
    • Sort diagonals in the bottom-left triangle in descending order.
    • Sort diagonals in the top-right triangle in ascending order.

Time Complexity
  • Extracting and sorting each diagonal: O(n log n) per diagonal.
  • Total Complexity: O(n² log n).

Space Complexity
  • O(1) (no extra space is used apart from a few variables).

C++ Implementation


#include <vector>
#include <algorithm>
#include <iostream>

using namespace std;

class Solution {
public:
    vector<vector<int>> sortMatrix(vector<vector<int>>& grid) {
        int n = grid.size();

        // Sort diagonals in the bottom-left triangle (including main diagonal)
        for (int i = 0; i < n; i++) {
            vector<int> diag;
            int row = i, col = 0;

            // Extract the diagonal
            while (row < n && col < n) {
                diag.push_back(grid[row][col]);
                row++;
                col++;
            }

            // Sort the diagonal in descending order
            sort(diag.begin(), diag.end(), greater<int>());

            // Place the sorted diagonal back into the matrix
            row = i, col = 0;
            int index = 0;
            while (row < n && col < n) {
                grid[row][col] = diag[index++];
                row++;
                col++;
            }
        }

        // Sort diagonals in the top-right triangle
        for (int j = 1; j < n; j++) {
            vector<int> diag;
            int row = 0, col = j;

            // Extract the diagonal
            while (row < n && col < n) {
                diag.push_back(grid[row][col]);
                row++;
                col++;
            }

            // Sort the diagonal in ascending order
            sort(diag.begin(), diag.end());

            // Place the sorted diagonal back into the matrix
            row = 0, col = j;
            int index = 0;
            while (row < n && col < n) {
                grid[row][col] = diag[index++];
                row++;
                col++;
            }
        }

        return grid;
    }
};

int main() {
    Solution sol;
    vector<vector<int>> grid = {{1, 7, 3}, {9, 8, 2}, {4, 5, 6}};

    vector<vector<int>> sortedGrid = sol.sortMatrix(grid);

    for (auto row : sortedGrid) {
        for (int num : row) {
            cout << num << " ";
        }
        cout << endl;
    }
    return 0;
}

Frequently Asked Questions (FAQs)

1. Can we solve this problem in O(n²) time?

Not exactly, since sorting takes O(n log n). However, using bucket sort (if constraints allow) could make sorting nearly O(n).

2. Why did we use an unordered_map?

It helps in efficiently storing and retrieving diagonal elements using (i - j) as a unique key.

3. What is the difference between the brute force and optimized approaches?

The brute force approach sorts each diagonal individually, while the optimized approach uses a hash map to store and sort diagonals more efficiently.

4. How do we identify diagonals in the matrix?

Each diagonal can be uniquely identified by the value (i - j), where i is the row index and j is the column index.

5. Can this problem be solved for non-square matrices?

No, the problem specifically requires a square matrix (n × n).

6. What is the space complexity of the optimized approach?

The space complexity is O(n²) due to the hash map storing all diagonal elements.

7. Is there a way to further optimize this solution?

Using bucket sort (if applicable) could reduce the sorting time to O(n), but the overall complexity would still be O(n²).

8. Why do we sort diagonals in different orders?

Diagonals in the bottom-left triangle are sorted in descending order, while those in the top-right triangle are sorted in ascending order, as per the problem requirements.

9. Can this problem be solved using recursion?

Yes, but it would be less efficient due to the overhead of recursive calls.

10. What are the real-world applications of this problem?

This problem helps in understanding matrix manipulation and sorting techniques, which are useful in image processing, data analysis, and machine learning.
