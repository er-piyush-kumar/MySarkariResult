Leetcode Problem 2965: Find Missing and Repeated Values

In this article, we’ll solve Leetcode Problem 2965: Find Missing and Repeated Values. We’ll explore multiple approaches, including Brute Force, Mathematical Formula, Sorting, and XOR Bit Manipulation. We’ll also analyze the time and space complexity of each solution and answer some frequently asked questions.

Problem Statement

You are given a 0-indexed 2D integer matrix grid of size n * n with values in the range [1, n2]. Each integer appears exactly once except for one number that appears twice and another number that is missing. Your task is to find the repeating and missing numbers.

Example 1:


Input: grid = [[1,3],[2,2]]
Output: [2,4]
Explanation: Number 2 is repeated, and number 4 is missing.

Example 2:


Input: grid = [[9,1,7],[8,9,2],[3,4,6]]
Output: [9,5]
Explanation: Number 9 is repeated, and number 5 is missing.

Approach 1: Brute Force

The brute force approach involves using a hash map to count the occurrences of each number in the grid. We then iterate through the map to find the repeating and missing numbers. 


class Solution {
public:
    vector findMissingAndRepeatedValues(vector>& grid) {
        vector ans(2);
        map mp;

        // Count occurrences of each number
        for (auto &v : grid) {
            for (int i = 0; i < v.size(); i++) {
                mp[v[i]]++;
            }
        }

        // Find repeating and missing numbers
        int i = 0;
        for (auto &e : mp) {
            i++;
            if (e.second > 1) ans[0] = e.first; // Repeating number
            if (e.first != i && ans[1] == 0) ans[1] = i; // Missing number
        }
        if (ans[1] == 0) ans[1] = i + 1; // Edge case for missing number

        return ans;
    }
};

Time Complexity: O(n2) because we iterate through the grid and the map.
Space Complexity: O(n2) due to the hash map.

Approach 2: Mathematical Formula

This approach uses mathematical formulas to find the repeating and missing numbers. We calculate the sum and sum of squares of the grid elements and compare them with the expected values. 


class Solution {
public:
    vector findMissingAndRepeatedValues(vector>& grid) {
        vector ans(2);
        long long n = grid.size();
        n *= n;

        long long gridSum = 0, exSum = (n * (n + 1)) / 2;
        long long gridSqSum = 0, exSqSum = (n * (n + 1) * (2 * n + 1)) / 6;

        // Calculate grid sum and grid square sum
        for (int i = 0; i < grid.size(); i++) {
            for (int j = 0; j < grid[0].size(); j++) {
                gridSum += grid[i][j];
                gridSqSum += (grid[i][j] * grid[i][j]);
            }
        }

        int eq1 = exSum - gridSum; // (a - b)
        int eq2 = (gridSqSum - exSqSum) / (exSum - gridSum); // (a + b)

        ans[0] = abs((eq1 + eq2) / 2); // Repeating number
        ans[1] = (eq2 - eq1) / 2; // Missing number

        return ans;
    }
};

Time Complexity: O(n2) due to grid traversal.
Space Complexity: O(1) as we only use a few variables.

Approach 3: Using Sorting

This approach involves sorting the grid elements and then iterating through the sorted list to find the repeating and missing numbers. 


class Solution {
public:
    vector findMissingAndRepeatedValues(vector>& grid) {
        vector ans(2);
        int n = grid.size();
        vector flattened;

        // Flatten the 2D grid into a 1D array
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                flattened.push_back(grid[i][j]);
            }
        }

        // Sort the flattened array
        sort(flattened.begin(), flattened.end());

        // Find repeating and missing numbers
        for (int i = 0; i < flattened.size() - 1; i++) {
            if (flattened[i] == flattened[i + 1]) {
                ans[0] = flattened[i]; // Repeating number
            }
            if (flattened[i + 1] != flattened[i] + 1) {
                ans[1] = flattened[i] + 1; // Missing number
            }
        }

        // Handle edge case for missing number
        if (ans[1] == 0) {
            ans[1] = n * n;
        }

        return ans;
    }
};

Time Complexity: O(n2 log n) due to sorting.
Space Complexity: O(n2) for the flattened array.

Approach 4: Using XOR Bit Manipulation

This approach uses XOR to find the repeating and missing numbers. XOR has the property that it cancels out duplicate numbers, making it useful for finding mismatches. 


class Solution {
public:
    vector findMissingAndRepeatedValues(vector>& grid) {
        vector ans(2);
        int n = grid.size();
        int totalSize = n * n;

        int xorAll = 0;

        // XOR all elements in the grid and numbers from 1 to n^2
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                xorAll ^= grid[i][j];
            }
        }
        for (int i = 1; i <= totalSize; i++) {
            xorAll ^= i;
        }

        // Find the rightmost set bit
        int rightmostSetBit = xorAll & (-xorAll);

        int group1 = 0, group2 = 0;

        // Partition numbers into two groups based on the rightmost set bit
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] & rightmostSetBit) {
                    group1 ^= grid[i][j];
                } else {
                    group2 ^= grid[i][j];
                }
            }
        }
        for (int i = 1; i <= totalSize; i++) {
            if (i & rightmostSetBit) {
                group1 ^= i;
            } else {
                group2 ^= i;
            }
        }

        // Determine which is the repeating and missing number
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == group1) {
                    ans[0] = group1;
                    ans[1] = group2;
                    return ans;
                }
            }
        }

        ans[0] = group2;
        ans[1] = group1;
        return ans;
    }
};

Time Complexity: O(n2) due to grid traversal.
Space Complexity: O(1) as we only use a few variables.

FAQs

1. What is the brute force approach?

The brute force approach uses a hash map to count occurrences of each number and then identifies the repeating and missing numbers.

2. Why is the mathematical formula approach more efficient?

It avoids extra space by using mathematical properties to find the solution.

3. What is the time complexity of the brute force approach?

O(n2) due to grid traversal and map operations.

4. What is the space complexity of the mathematical approach?

O(1) as it uses only a few variables.

5. Can this problem be solved in O(n) time?

No, because we need to traverse the entire grid, which takes O(n2) time.
