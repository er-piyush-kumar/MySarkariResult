Leetcode 2379 | Minimum Recolors to Get K Consecutive Black Blocks

The problem "Minimum Recolors to Get K Consecutive Black Blocks" is a classic example of a sliding window problem. It requires us to find the minimum number of recoloring operations needed to achieve at least k consecutive black blocks in a given string of blocks. In this article, we’ll explore multiple approaches to solve this problem, including a Brute Force method, an Optimized Sliding Window approach, and an In-Place Counting approach. We’ll also analyze the time and space complexity of each solution and answer some frequently asked questions.

Problem Link : Minimum Recolors to Get K Consecutive Black Blocks

Understanding the Problem

We are given a string blocks of length n, where each character is either 'W' (white) or 'B' (black). We are also given an integer k, which represents the desired number of consecutive black blocks. In one operation, we can recolor a white block to black. The goal is to find the minimum number of operations required to achieve at least one occurrence of k consecutive black blocks.

Example 1


Input:
blocks = "WBBWWBBWBW", k = 7

Output:
3

Explanation:
One way to achieve 7 consecutive black blocks is to recolor the 0th, 3rd, and 4th blocks
so that blocks = "BBBBBBBWBW". 
It can be shown that there is no way to achieve 7 consecutive black blocks in less than 3 operations.
Therefore, we return 3.

Example 2


Input:
blocks = "WBWBBBW", k = 2

Output:
0

Explanation:
No changes need to be made, since 2 consecutive black blocks already exist.
Therefore, we return 0.

Brute Force Approach – Checking All Possible Substrings

The brute force approach involves checking all possible substrings of length k in the given string and calculating the number of white blocks in each substring. The minimum number of white blocks across all substrings is the answer.

Step-by-Step Approach
  1. Iterate Through the String: For each starting index i in the string, consider the substring of length k starting at i.
  2. Count White Blocks: Count the number of white blocks ('W') in the current substring.
  3. Track Minimum Operations: Keep track of the minimum number of white blocks across all substrings.
  4. Return the Result: The minimum count of white blocks is the answer.

Time Complexity
  • Iterating through the string: O(n)
  • Counting white blocks in each substring: O(k)
  • Total Complexity: O(n * k)

Space Complexity
  • O(1) (no extra space is used apart from a few variables).

C++ Implementation


#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    int minimumRecolors(string blocks, int k) {
        int n = blocks.size();
        int minOps = INT_MAX;

        for (int i = 0; i <= n - k; i++) {
            int countW = 0;
            for (int j = i; j < i + k; j++) {
                if (blocks[j] == 'W') {
                    countW++;
                }
            }
            minOps = min(minOps, countW);
        }

        return minOps;
    }
};

Drawbacks of the Brute Force Approach
  • It is inefficient for large inputs due to the nested loop structure.
  • It does not take advantage of overlapping substrings.

Optimized Approach – Sliding Window

The sliding window approach optimizes the brute force method by reusing the count of white blocks from the previous window. This reduces the time complexity significantly.

Steps to Optimize
  1. Initialize the Window: Count the number of white blocks in the first window of size k.
  2. Slide the Window: Move the window one step at a time and update the count of white blocks by considering the new element added and the element removed from the window.
  3. Track Minimum Operations: Keep track of the minimum number of white blocks across all windows.
  4. Return the Result: The minimum count of white blocks is the answer.

Time Complexity
  • Initializing the first window: O(k)
  • Sliding the window: O(n - k)
  • Total Complexity: O(n)

Space Complexity
  • O(1) (no extra space is used apart from a few variables).

C++ Implementation


#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    int minimumRecolors(string blocks, int k) {
        int n = blocks.size();
        int countW = 0;
        int minOps = INT_MAX;

        // Initialize the first window
        for (int i = 0; i < k; i++) {
            if (blocks[i] == 'W') {
                countW++;
            }
        }
        minOps = countW;

        // Slide the window
        for (int i = k; i < n; i++) {
            if (blocks[i - k] == 'W') {
                countW--;
            }
            if (blocks[i] == 'W') {
                countW++;
            }
            minOps = min(minOps, countW);
        }

        return minOps;
    }
};

Approach 3: In-Place Counting

This approach is similar to the sliding window method but avoids using extra variables to store the count of white blocks. Instead, it directly updates the minimum operations while iterating through the string.

Steps to Implement
  1. Initialize Counters: Use two pointers to represent the start and end of the window.
  2. Iterate Through the String: Move the end pointer and update the count of white blocks in the current window.
  3. Update Minimum Operations: If the window size reaches k, update the minimum operations and move the start pointer.
  4. Return the Result: The minimum count of white blocks is the answer.

Time Complexity
  • Iterating through the string: O(n)
  • Total Complexity: O(n)

Space Complexity
  • O(1) (no extra space is used apart from a few variables).

C++ Implementation


#include <string>
#include <algorithm>

using namespace std;

class Solution {
public:
    int minimumRecolors(string blocks, int k) {
        int n = blocks.size();
        int countW = 0;
        int minOps = INT_MAX;
        int start = 0;

        for (int end = 0; end < n; end++) {
            if (blocks[end] == 'W') {
                countW++;
            }

            if (end - start + 1 == k) {
                minOps = min(minOps, countW);
                if (blocks[start] == 'W') {
                    countW--;
                }
                start++;
            }
        }

        return minOps;
    }
};

Frequently Asked Questions (FAQs)

1. Can we solve this problem in O(n) time?

Yes, the sliding window and in-place counting approaches solve the problem in O(n) time.

2. Why is the sliding window approach more efficient?

It reuses the count of white blocks from the previous window, reducing redundant calculations.

3. What is the difference between the brute force and sliding window approaches?

The brute force approach checks each substring independently, while the sliding window approach reuses information from the previous window.

4. How do we identify the window in the sliding window approach?

The window is identified by its start and end indices, which are updated as the window slides through the string.

5. Can this problem be solved for non-consecutive blocks?

No, the problem specifically requires consecutive blocks.

6. What is the space complexity of the sliding window approach?

The space complexity is O(1) because it uses a constant amount of extra space.

7. Is there a way to further optimize this solution?

The sliding window approach is already optimal with O(n) time complexity.

8. Why do we need to track the minimum number of white blocks?

Because the minimum number of white blocks corresponds to the minimum number of recoloring operations needed.

9. Can this problem be solved using recursion?

Yes, but it would be less efficient due to the overhead of recursive calls.

10. What are the real-world applications of this problem?

This problem helps in understanding string manipulation and sliding window techniques, which are useful in data analysis and pattern recognition.
